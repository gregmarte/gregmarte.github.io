Iterative Development



Figuring out how to do projects in an iterative way is essential to building value.
A project could take twice as long and still be comparable to a waterfall project.

Any opportunity to break a task down into a value creating piece is essential to modern project design.
Even if reworking at each step will be double the time cost.

Also, with iterative development we gain all of these benefits:

  • Learn and adjust development through each planning stage
  • Move into the unknown, while managing the risk
  • Tighter team integration (, at the risk of more confusion, and more critical paths in the project)*
  • Able to halt a project at any point, and move onto another higher value proposition

*Teams do need to learn to operate iteratively.
